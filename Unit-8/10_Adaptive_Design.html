In contrast to responsive design that fluidly rearranges the same layout of elements on a web page based on the user's screen size, adaptive design uses static layouts for specific screen sizes. To create adaptive web design, it is important to first understand the concepts of the viewport and different breakpoints for the different screen sizes your web page may be viewed on. What is the viewport and why is it important in web design? The viewport is the area on a web page that is visible to the user. In the past (pre-mobile phone days) people viewed websites on desktop size computer monitors. As most of these monitor screens were relatively the same dimensions, a web designer could code their layout to these dimensions. With the invention of and widespread use of smartphones and tablets, designers had to decide how their web pages would appear on all different types and sizes of screens. Just scaling down or 'shrinking' the entire web page layout was not considered an ideal solution. Web designers wanted the ability to have their web page layouts adapt to particular viewports. They wanted to use different layouts for the 'same' web page. Which layout a user sees is designed for the device and viewport the user is using. How do you use the viewport? Use the tag to control the viewport and in doing so, the dimensions and scaling of the web page. Include the following code in all your web pages: html Copy name="viewport" content="width-device-width, initial-scale=1.0"> The value for content width-device-width tells the browser to set the width of your web page to the screen width your web page is being displayed on. The initial-scale=1.0 part of the code above tells the browser to set the initial zoom level of your web page to 1.0 when the browser first loads your web page. Breakpoint A breakpoint is the point at which a layout adapts (changes) based on the width of the screen on the user's device. Breakpoints are sometimes referred to as media query breakpoints as media query is used when setting these breakpoints. You may want different layouts for different size screens.
